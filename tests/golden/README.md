# Golden Output Snapshots

This directory contains golden JSON output files produced by the deterministic
Python pipeline modules when run against their test fixtures. These files serve
as the ground truth for pytest regression assertions.

## Regeneration

Golden files are regenerated by re-running the relevant pipeline modules and
copying their output back into this directory. Each module is idempotent:
re-running against the same fixture produces byte-identical output.

## Naming Convention

Each golden file follows the pattern `<script-name>--<fixture-name>.json`.
For multi-input modules, `<fixture-name>` reflects the primary input or
directory name.

## Excluded Modules

| Module | Reason |
|--------|--------|
| `seo_pipeline/serp/fetch_serp.py` | Requires live DataForSEO API calls; cannot produce deterministic output offline. |
| `seo_pipeline/keywords/fetch_keywords.py` | Requires live DataForSEO API calls; cannot produce deterministic output offline. |
| `seo_pipeline/extractor/extract_page.py` | Requires live HTTP requests to crawl pages; cannot produce deterministic output offline. Existing page fixture JSONs serve as static snapshots instead. |

## Non-Deterministic Field Handling

The `assemble_briefing_data` output contains two non-deterministic fields that
are frozen to fixed sentinel values during golden file generation:

- `meta.phase1_completed_at` is frozen to `"2026-01-01T00:00:00.000Z"`
- `meta.current_year` is frozen to `2026`

Tests compare against these sentinels rather than live timestamps.
